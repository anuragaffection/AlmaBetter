Answer

01.
Using the try...catch statement

Using the try...catch statement is a common way to handle errors when making API requests in React, allowing you to gracefully handle errors and prevent crashes.





02.
Cross-Origin Resource Sharing

CORS (Cross-Origin Resource Sharing) is a security feature implemented by web browsers that allows or restricts web pages in one domain from making requests to a different domain.





03.
Creating responsive layouts

The Bootstrap grid system is used to create responsive and flexible page layouts.




04.
A function passed as an argument to another function

In Node.js, callback functions are functions passed as arguments to other functions and are commonly used for asynchronous operations.




05.
A set of multiple MongoDB servers that maintain copies of the same data.

A replica set in MongoDB is a group of multiple MongoDB servers that work together to ensure high availability and data redundancy. Each server in the replica set maintains a copy of the same data, and if one server goes down, the others can continue serving data, providing fault tolerance.





06.
MongoDB uses a document-oriented data model.

MongoDB is a NoSQL database that uses a document-oriented data model to store data in flexible, schema-less documents, typically in BSON format (Binary JSON). Unlike traditional relational databases, MongoDB does not use tables with fixed schemas, and it does not support complex JOIN operations.







07.
Application Programming Interface

API stands for Application Programming Interface. It is a set of rules and protocols that allow different software applications to communicate with each other.

The HTTP GET method is typically used for retrieving data from a server in a RESTful API.

200 OK

The HTTP status code 200 OK indicates that the server has successfully processed the request.

MAX()

The MAX() function is used to find the maximum value in a column.






11.
Structuring a database to minimize data duplication and ensure data integrity.

Normalization is the process of organizing a database to reduce data redundancy and maintain data integrity by defining relationships between tables.

18.
It helps React identify which items have changed, added, or removed when re- rendering a list.

The "key" prop is used in React to help it identify which items in a list have changed, been added, or removed when re-rendering. It's crucial for efficient updates of lists in React.







19.
To define routes and handlers for specific URL paths in an Express application.

The Express Router is used to define routes and their corresponding handlers for specific URL paths in an Express application. It helps organize the code and manage different endpoints by grouping them based on their functionality.






20.
A way to distribute data across multiple servers to improve scalability.

Sharding is a MongoDB technique used to horizontally partition data across multiple servers or nodes to improve scalability and distribute the workload. It allows MongoDB to handle large amounts of data and high write and read loads.

useEffect hook is commonly used in React to perform side effects, such as making API requests, when a component renders or updates.

A unique identifier for each row in the table.

23.
Structured Query Language

SQL stands for Structured Query Language, which is a domain-specific language used for managing and querying relational databases.






24.
A function that handles requests and responses in the middleware stack.

In Express.js, middleware are functions that are executed in the request-response cycle. They have access to the request object (req), the response object (res), and the next middleware function in the stack. Middleware can be used for tasks like authentication, logging, and error handling.
